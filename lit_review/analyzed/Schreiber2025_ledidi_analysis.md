## 1. Summary

Ledidi is a computational method that frames the design of genomic edits as a continuous optimization problem. Given an initial DNA sequence and one or more pre-trained ML models, Ledidi uses Gumbel-softmax reparameterization to learn a weight matrix over nucleotide positions, then samples discrete edited sequences that achieve a desired model output while minimizing the number of edits (via an input loss term). The method can optimize against multiple models simultaneously to design edits with multiple desired characteristics (e.g., cell type-specific enhancers with controllable TF binding patterns). The paper introduces "affinity catalogs" — sets of designs produced by sweeping across a spectrum of target outcomes — which provide a resource for understanding how sequence features relate to function. Demonstrated on TF binding (BPNet), chromatin accessibility (ChromBPNet), transcription (ProCapNet), and enhancer activity (DeepSTARR) models.

## 2. Relevance to PoolParty Manuscript

Ledidi and PoolParty address fundamentally different problems:

- **Ledidi** designs individual optimized sequences by gradient-based editing of a template to achieve specific ML model predictions. Output: one (or a few) optimized sequences.
- **PoolParty** designs libraries of many variant sequences for systematic exploration (MPRA, DMS, in silico experiments). Output: thousands to millions of sequences with structured metadata.

The "affinity catalog" concept is the closest conceptual connection — sweeping target values produces a set of designed sequences. However, these are generated by independent optimization runs, not by composable combinatorial logic, and they lack the structured metadata (design cards, names) that PoolParty provides.

| Feature | Ledidi | PoolParty |
|---------|--------|-----------|
| Purpose | Design minimal edits for desired ML output | Design variant sequence libraries |
| Approach | Gradient-based optimization | Composable DAG of Operations |
| Output | Individual optimized sequence(s) | Libraries of thousands–millions of variants |
| ML model required | Yes (core requirement) | No (model-agnostic) |
| Design cards | No | Core feature |
| Combinatorial design | No | Core feature |
| Mutagenesis | No (optimization, not systematic variation) | 20+ built-in Operations |

## 3. Citation Recommendations

**Should not be cited in the Introduction's related-work section.** Ledidi is not a library design tool — it is a sequence optimization method for ML-guided design of individual sequences. It does not belong in any of the three categories of tools discussed in the Introduction (general-purpose toolkits, assay-specific tools, sequence optimizers for synthesis constraints).

**Optional mention in the Discussion** as an example of ML-guided sequence design that is complementary to PoolParty's systematic approach. PoolParty creates structured libraries for probing model behavior (as demonstrated in the SpliceAI example); Ledidi uses models to design individual sequences with desired properties. These are opposite directions of the same interaction between libraries and models.

## 4. Novelty Assessment

**No threat to novelty claims.** Completely different problem domain (ML-guided optimization vs. systematic library design). No architectural overlap. No special precautions needed.
